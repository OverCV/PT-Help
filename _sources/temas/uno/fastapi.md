~~~{dropdown} M√°s sobre versionamiento!

Cuando defines las dependencias en tu archivo `requirements.txt`, puedes especificar las versiones de varias maneras para asegurar que tu proyecto use versiones compatibles de cada librer√≠a. Aqu√≠ te explico algunas opciones:

    Esto significa que `fastapi` se instalar√° en la versi√≥n `0.68.0`.

    ```
    Esto significa que `uvicorn` se instalar√° en la versi√≥n `0.15.0` o una versi√≥n m√°s reciente.

    ```text
    pydantic<1.8.2
    ```
    Esto significa que `pydantic` se instalar√° en una versi√≥n anterior a la `1.8.2`.

    ```text

    Esto significa que `uvicorn` se instalar√° en una versi√≥n entre la `0.15.0` y la `0.17.0`, inclusive.

    Esto significa que `ruff` se instalar√° en una versi√≥n compatible con la `0.1.0`. Generalmente, esto incluye cualquier versi√≥n que no cambie el primer n√∫mero despu√©s del punto (por ejemplo, `0.1.1`, `0.1.2`, pero no `0.2.0`).

## FastAPI Web Framework

FastAPI nos permite hacer desarrollos en la web usando Python, no obstante tiene dependencia de `uvicorn` para poder ejecutar el servidor. Para ello, vamos a crear un archivo llamado `main.py` en la carpeta ra√≠z y vamos a agregar el siguiente c√≥digo:

```python

app: FastAPI = FastAPI(
    title="Mini Backend | Mi Nombre.",
    summary="T√©cnicas de programaci√≥n, 2024B",
    version="1.0.0",
)


@app.get("/")
def root() -> dict[str, str]:
    return {"data": "Hello algorithms!"}
```

```{dropdown} Directorio actual
```markdown
Mini-Back

‚îî‚îÄ‚îÄ requirements.txt
```

Para ver el resultado abrimos una terminal y ejecutamos el siguiente comando:

```powershell
uvicorn main:app --reload
```

```{note}
El comando `uvicorn` nos permite ejecutar el servidor web, `main:app` indica que en el archivo `main.py` contiene el objeto o instancia de `app` FastAPI *(el aplicativo)* y `--reload` permite recargar el servidor cada vez que se realice un cambio al c√≥digo.
```

De esta forma, cuando abramos un navegador y vayamos a la direcci√≥n indicada, cual por defecto es `http://127.0.0.1:8000/`, la previamente mencionada direcci√≥n IP de Localhost, donde lo que precede a los dos puntos es el puerto *(8080)*.

Deber√≠amos ver el mensaje `{"data": "Hello algorithms!"}` en el navegador.

```{note}
Para detener el servidor, basta con presionar `ctrl + c` en la terminal.
```

Pero entonces, ¬øQu√© hemos hecho? Hemos creado un servidor web que nos devuelve un mensaje en formato JSON *(JavaScript Object Notation)*, en este caso `{"data": "Hello algorithms!"}` es un diccionario con clave `data` y valor `Hello algorithms!`, pero se convierte en JSON en el proceso de respuesta de forma forzosa. Ahora, la direcci√≥n a la que accedemos despu√©s del puerto `8080` la hemos dejado en blanco en nuestro navegador, adem√°s, hemos definido que ante una petici√≥n de tipo `GET` a la direcci√≥n ra√≠z `/`, se ejecute la funci√≥n `root`, es por esto que nos devuelve el mensaje mencionado.

FastAPI nos permite hacer desarrollos web de manera r√°pida y eficiente, adem√°s de ser f√°cil de aprender y usar.

### An√°lisis de B√∫squeda Secuencial

La b√∫squeda secuencial o lineal es un algoritmo donde dado un arreglo desordenado o no de elementos distintos, valida la existencia de un elemento dado por el usuario *(la precondici√≥n)*, si est√° devuelve el √≠ndice en el que se encuentre, si no, devuelve `-1` *(la postcondici√≥n)*. Este algoritmo es de complejidad computacional temporal $T(n)\in O(n)$ *(lineal)*, es decir, su tiempo de ejecuci√≥n es proporcional al tama√±o del arreglo. en el peor escenario.

```{note}
El argumento `dto` es una abreviaci√≥n de `Data Transfer Object`, es una forma de referirse a un objeto que se utiliza para transferir datos entre subsistemas de una aplicaci√≥n. En este caso, `dto` es el objeto que recibe la petici√≥n del Cliente y es el objeto que se env√≠a como respuesta al Cliente.
```

Se manejar√° la siguiente entrada de datos:
<img src='../../_static/images/tema_01/requesting.png' style='border-radius: 1rem;'/>

```python
def sequential_search(dto: SolicitudBusqueda) -> RespuestaBusqueda:
    indice_sin_encontrar: int = -1
    for indice, entero in enumerate(dto.arreglo): # (idx, elem) <- enumerate(colection)
        if entero == dto.objetivo:
            return RespuestaBusqueda(
                indice=indice,
                encontrado=True,
            )
    return RespuestaBusqueda(
        indice=indice_sin_encontrar,
        encontrado=False,
    )
```

Para la resoluci√≥n del problema haremos uso de un ciclo `for-each` simult√°neo a uno `for-i`, en Python existe la funci√≥n `enumerate` que nos permite recorrer una colecci√≥n asociando un indice a cada elemento, nos devuelve una tupla donde primero obtenemos el √≠ndice y luego el elemento.
En cada iteraci√≥n comparamos si el elemento es igual al buscado, si es as√≠ podemos devolver el √≠ndice e indicar que se ha encontrado con `True`, caso contrario a no encontrarlo, devolvemos `-1` e indicamos que no se ha encontrado con `False`.
